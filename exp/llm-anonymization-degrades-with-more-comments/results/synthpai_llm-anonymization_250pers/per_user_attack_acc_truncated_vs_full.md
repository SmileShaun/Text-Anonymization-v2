# Per-user Attack Acc: truncated vs full (top-1 / top-3)

同一批 250 个 llm-anonymization pers；按 **full comments 条数降序**。

- **原始** = level 0；**匿名** = level 3
- **top-1** = 第一个 guess 正确；**top-3** = 前三个 guess 任一正确
- **截断** ≤25 条；**全量** 不截断
- 单元格格式：`top1 / top3`

共 250 人。明细 CSV：`per_user_attack_acc_truncated_vs_full.csv`。

## 结论：全量·匿名 top-3 vs 截断·匿名 top-3

比较 **全量·匿名 (L3, top-3)** 与 **截断·匿名 (L3, top-3)**：

- **并非全体都显著更高。** 全体 micro：trunc 72.7% vs full 75.2%（仅 **+2.6 pp**）。
- **当 comments 明显更多时，full 匿名后的 top-3 往往更高**（更多文本带来更多残留可攻击线索）：
  - `full > trunc` 的 34 人（宏平均）：71.2% → 77.0%（**+5.9 pp**；15 升 / 3 降 / 16 平）
  - `#full ≥ 30`：72.1% → 79.3%（**+7.3 pp**）
  - `#full ≥ 40`：70.5% → 85.7%（**+15.2 pp**；8 升 / 0 降）
  - 分桶 `#full ∈ [36,51]`：69.2% → 85.0%（**+15.9 pp**）
- **comments 不多或两边一样长时**，差异小，且受独立推理随机性影响，谈不上稳定显著更高（例如分桶 16–25、26–35 时 full 甚至略低或接近）。

一句话：**comments 多、且 full 明显长于 trunc 时，full·匿名 top-3 往往显著高于 trunc·匿名 top-3；否则趋势不明显。**
